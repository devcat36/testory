# 비기능 요구사항 (Non-Functional Requirements)

## 1. 호스팅 / 비용

| ID | 우선순위 | 요구사항 |
|---|---|---|
| NFR-1 | MUST | 백엔드 서버 없이 동작하는 **정적 사이트**로 빌드·배포한다. |
| NFR-2 | MUST | 상시 서버 비용 ₩0 (정적 호스팅/CDN 범위 내). |
| NFR-3 | SHOULD | OG 이미지 동적 생성이 필요하면 그 부분만 엣지/서버리스 함수로 한정한다. |
| NFR-4 | COULD | 배포 타깃은 추후 결정(정적 호스트 또는 홈랩 HA). 지금은 호스트 비종속으로 설계한다. |

## 2. 성능

| ID | 우선순위 | 요구사항 |
|---|---|---|
| NFR-5 | MUST | **모바일 우선**. 첫 화면이 모바일 4G에서 2.5초 이내 인터랙션 가능(LCP ≤ 2.5s 목표). |
| NFR-6 | SHOULD | 질문 전환은 네트워크 왕복 없이 즉시(클라이언트 라우팅). |
| NFR-7 | SHOULD | 결과/공유 카드 이미지는 캐싱하여 재요청 시 빠르게 제공한다. |

## 3. 호환성

| ID | 우선순위 | 요구사항 |
|---|---|---|
| NFR-8 | MUST | **카카오톡 인앱 브라우저**에서 정상 동작한다(공유 후 유입의 주 경로). |
| NFR-9 | MUST | 최신 모바일 Safari/Chrome 및 데스크톱 Chrome에서 동작한다. |
| NFR-10 | SHOULD | 반응형 — 모바일/태블릿/데스크톱에서 레이아웃이 깨지지 않는다. |

## 4. 공유 미리보기 (SEO/OG)

| ID | 우선순위 | 요구사항 |
|---|---|---|
| NFR-11 | MUST | 결과 URL은 OpenGraph/Twitter 카드 메타를 갖춰 카톡·SNS에서 썸네일+제목 미리보기가 뜬다. |
| NFR-12 | SHOULD | 시작 URL에도 매력적인 기본 OG 카드를 둔다. |

## 5. 개인정보 / 데이터

| ID | 우선순위 | 요구사항 |
|---|---|---|
| NFR-13 | MUST | 개인정보를 수집·전송하지 않는다(답변·결과는 클라이언트에서만 처리). |
| NFR-14 | SHOULD | 분석 도구를 붙이더라도 익명 집계 수준으로 제한한다. |

## 6. 언어 / 콘텐츠

| ID | 우선순위 | 요구사항 |
|---|---|---|
| NFR-15 | MUST | 1차 언어는 한국어. 카피 톤은 "재미 + 따뜻함 + 약간의 정곡 찌르기". |
| NFR-16 | COULD | 다국어(i18n) 확장 가능하도록 문자열을 분리한다. |

## 7. 유지보수

| ID | 우선순위 | 요구사항 |
|---|---|---|
| NFR-17 | SHOULD | 콘텐츠 수정(질문/캐릭터/카피)이 코드 변경 없이 데이터 파일 편집만으로 가능해야 한다. |
| NFR-18 | SHOULD | 점수 로직 불변식(동점 0·8유형 도달·희귀도 합 100%)을 검증하는 테스트를 둔다. |
