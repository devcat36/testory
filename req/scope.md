# 범위 (Scope)

## MVP (v1) — "링크 하나로 도는 테스트"

핵심 한 바퀴를 완성하는 것이 목표: **시작 → 21문항 → 결과 카드 → 공유**.

포함 (MUST):
- 시작 화면 (FR-1)
- 21문항 퀴즈 + 진행률 (FR-4~9)
- 클라이언트 채점 → 8유형 매핑 (FR-10~13)
- 결과 카드 화면 + 희귀도 (FR-14~15)
- 결과 고유 URL `/r/[code]` + OG 미리보기 (FR-16~17)
- 공유 동작(최소 링크 복사 + 공유 카피) (FR-20~21)
- 정적 빌드 / 서버 0 / 모바일 우선 / 카톡 인앱 동작 (NFR-1, 2, 5, 8, 11)

목표: 데이터(`test-design.json`) 기반으로 동작하는 단일 테스트 1종.

## 비범위 (Out of scope, v1에서 제외)

- AI/LLM 기반 개인화 분석 (정해진 분기로 충분 — discovery 결정)
- 사용자 계정 / 로그인 / 결과 저장 서버
- 결제 / 유료 상세 리포트
- 여러 테스트 동시 운영 (데이터 구조는 확장 가능하게 두되, v1은 1종)
- 관리자 페이지 / CMS

## 완료 기준 (Definition of Done, MVP)

- [ ] 모바일에서 시작→결과까지 끊김 없이 진행된다.
- [ ] 8개 유형 모두 실제로 도달 가능하다(점수 로직 테스트 통과).
- [ ] 결과 링크를 카톡에 붙이면 캐릭터 썸네일 카드가 미리보기로 뜬다.
- [ ] 서버 없이 정적 배포로 동작한다.
